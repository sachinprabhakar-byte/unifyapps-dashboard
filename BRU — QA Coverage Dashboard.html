BU_UC01-8 UC01 Status shows 'Under Review for Contract Type' when request is approved and contract created. Should show as completed, not requiring action. New Ask Bug The system is displaying a misleading status — showing 'Under Review' when the contract is actually completed. This is incorrect status tracking behavior. The BRD covers contract status updates throughout all 3 phases. Showing wrong status is a defect.
BU_UC01-12 UC01 On browser refresh - the same tab needs to be opened in UC01 UI. New Ask Bug Preserving tab state on browser refresh is standard expected UX behavior. Losing the user's position on refresh is a defect in the application, not a new feature request.

New Ask Bug The BRD specifies that validation results are posted to the discussion section and 'AI suggestions will be given to showcase data validation details.' Posting unnecessary content when there are no issues is a logic defect in the existing validation output — the system should only flag problems, not confirmations.
BU_UC01-20 UC01 Please only list issues in the discussion section. There's no need to list successful items, just to keep things shorter. New Ask Bug Same as BU_UC01-19. The BRD says the RPA should 'Inform the Contract Manager about the results based on schema field.' Listing successful validations alongside failures clutters the output and defeats the purpose. This is a defect in the validation output logic.
BU_UC01-24 UC01 Request 1282312 - Flagged for no attachments, but there were 2 Word Documents attached. New Ask Bug The system incorrectly flags a request as having no attachments when it clearly has 2 Word Documents. This is a validation logic error — the attachment detection is broken. This is unambiguously a Bug, not a New Ask.

New Ask Bug The BRD explicitly requires discrepancy identification and documentation (Step 7D: 'Calculate variance amounts', Step 11: 'Review and research any unexplained variances'). The Error Handling & Monitoring Framework (Section 8) further mandates discrepancy surfacing. Without a discrepancy tab, the system fails to meet these BRD requirements.
